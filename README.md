# 반짝국어 — AI 지문 분석 작업실

**반짝국어**는 국어 비문학·문학 지문을 입력하면 OpenAI API를 이용해 지문을 정돈하고, 핵심 내용을 요약하고, 글의 구조를 시각화하고, 형광펜으로 주요 구절을 표시하고, 마인드맵과 문제 풀이 환경까지 제공하는 학습용 정적 웹앱입니다. 단순히 “요약만 해 주는 도구”가 아니라, 사용자가 실제로 지문을 읽고 문제를 풀고 오답을 확인하고, 이해가 안 되는 부분을 AI에게 다시 질문하는 흐름을 목표로 만들어졌습니다.

이 README는 처음 사용하는 사람이 앱을 실행하고, API 키를 넣고, 지문을 정리하고, 분석하고, 형광펜과 마인드맵을 보고, 문제를 풀고, 저장하고, GitHub Pages로 배포하는 과정까지 한 번에 따라 할 수 있도록 작성한 상세 튜토리얼입니다.

---

## 1. 이 앱으로 할 수 있는 일

반짝국어의 핵심 기능은 다음과 같습니다.

1. **국어 지문 입력**
   - 직접 타이핑한 지문, 인터넷에서 복사한 지문, OCR로 추출한 지문을 넣을 수 있습니다.
   - OCR 지문처럼 줄바꿈과 띄어쓰기가 뒤죽박죽인 글도 어느 정도 정리할 수 있도록 `OCR 정리`와 `AI 정돈` 기능을 제공합니다.

2. **AI 지문 분석**
   - 전체 주제와 요약을 생성합니다.
   - 문단별 핵심 내용을 정리합니다.
   - 글의 전개 구조를 타임라인으로 보여 줍니다.
   - 개념, 주장, 근거, 비교·대조, 예시, 주의해야 할 구절을 형광펜으로 표시합니다.
   - 헷갈릴 만한 문장을 골라 쉬운 말로 풀어 설명합니다.
   - 난이도를 5개 기준으로 평가하고, 종합 난이도를 5점 만점으로 표시합니다.

3. **인터랙티브 마인드맵**
   - 중심 개념에서 가지가 뻗는 방식으로 지문 내용을 정리합니다.
   - 노드를 드래그할 수 있습니다.
   - 빈 공간을 드래그하거나 휠클릭 드래그로 화면을 이동할 수 있습니다.
   - 마우스 휠이나 `＋`, `−` 버튼으로 확대·축소할 수 있습니다.
   - 노드를 클릭하면 해당 개념의 상세 설명을 확인할 수 있습니다.

4. **문제 생성**
   - 분석이 끝난 뒤 별도로 문제를 생성할 수 있습니다.
   - 5지선다형, OX형, 서술형 문제를 생성합니다.
   - 5지선다 문제는 처음부터 정답이 보이지 않습니다. 사용자가 먼저 선택한 뒤 `채점 및 정오 보기`를 누르면 정답과 해설이 열립니다.
   - OX 문제는 개별 채점과 전체 채점을 모두 지원합니다.
   - 서술형 문제는 직접 답안을 작성한 뒤 AI 채점을 받을 수 있습니다.

5. **AI 질문**
   - 문제를 풀다가 납득이 안 되는 선지나 개념이 있으면 문항별로 AI에게 질문할 수 있습니다.
   - 지문에서 특정 구절을 드래그한 뒤 메모 탭에서 AI 설명을 요청할 수 있습니다.

6. **저장과 복귀**
   - 분석 결과를 저장하고 나중에 다시 열 수 있습니다.
   - 샘플 지문을 봤다가 다시 `내 지문`으로 돌아와도 기존 분석 결과와 문제 풀이 상태가 유지됩니다.
   - `새 분석 노트` 버튼으로 완전히 새로운 작업을 시작할 수 있습니다.

## 2. 실행 방법

이 앱은 정적 웹앱입니다. 별도의 백엔드 서버 없이 브라우저에서 실행됩니다. 다만 `index.html`을 그냥 더블클릭해서 열면 모듈 파일을 제대로 불러오지 못하는 경우가 있으므로, 간단한 로컬 서버로 실행하는 것을 권장합니다.

### Windows에서 실행하기

1. 이 저장소 또는 ZIP 파일을 다운로드합니다.
2. 압축을 풉니다.
3. `index.html`이 들어 있는 폴더를 엽니다.
4. 폴더 주소창에 `cmd`를 입력하고 Enter를 누릅니다.
5. 아래 명령어를 입력합니다.

```bash
py -m http.server 8123
```

또는 Python 명령이 `python`으로 잡혀 있다면 다음 명령을 사용할 수도 있습니다.

```bash
python -m http.server 8123
```

6. 브라우저에서 아래 주소로 접속합니다.

```text
http://127.0.0.1:8123
```

7. 예전 버전을 열었던 적이 있다면 반드시 `Ctrl + F5`로 강력 새로고침하세요. 브라우저 캐시에 이전 JavaScript 파일이 남아 있으면 수정된 기능이 반영되지 않을 수 있습니다.

### macOS / Linux에서 실행하기

터미널에서 프로젝트 폴더로 이동한 뒤 다음 명령어를 실행합니다.

```bash
python3 -m http.server 8123
```

브라우저에서 다음 주소를 엽니다.

```text
http://127.0.0.1:8123
```

### VS Code Live Server로 실행하기

Python 명령어가 어렵다면 VS Code의 Live Server 확장을 사용할 수도 있습니다.

1. VS Code를 설치합니다.
2. Extensions에서 `Live Server`를 설치합니다.
3. 프로젝트 폴더를 VS Code로 엽니다.
4. `index.html`을 우클릭합니다.
5. `Open with Live Server`를 누릅니다.

## 3. 처음 화면 사용법

앱에 들어가면 바로 분석 화면이 뜨지 않고, 인트로 화면이 먼저 표시됩니다. 이 화면은 앱의 목적을 간단히 보여 주는 시작 화면입니다.

- `시작하기`를 누르면 지문 분석 작업실로 이동합니다.
- 샘플을 먼저 보고 싶다면 상단의 `샘플` 버튼을 사용할 수 있습니다.
- 이미 분석해 둔 지문이 있다면 저장 탭에서 다시 불러올 수 있습니다.

인트로 화면은 단순 장식이 아니라, 실제 작업 화면으로 들어가기 전에 사용자가 “지금부터 분석을 시작한다”는 흐름을 만들어 주기 위한 화면입니다.

## 4. 메뉴 버튼 사용법

오른쪽 위의 `메뉴` 버튼을 누르면 슬라이드 패널이 열립니다. 이곳에는 분석 자체보다는 설정과 편의 기능이 모여 있습니다.

메뉴에서 설정할 수 있는 항목은 다음과 같습니다.

1. **OpenAI API Key**
   - 사용자의 OpenAI API 키를 입력합니다.
   - 키를 입력한 뒤 `키 적용`을 누르면 앱이 키를 인식합니다.
   - 키가 인식되면 화면에 `키 인식됨` 상태가 표시됩니다.

2. **일반 분석 모델**
   - 기본 지문 분석에 사용할 모델을 지정합니다.
   - 예: `gpt-4.1-mini`, `gpt-4.1`, `gpt-5.4` 등 사용자가 접근 가능한 모델명을 입력할 수 있습니다.

3. **전문 추론 모델**
   - 더 정밀한 분석이 필요할 때 사용할 모델입니다.
   - 전문 추론 모델을 사용하면 분석 품질이 좋아질 수 있지만 비용과 시간이 더 늘어날 수 있습니다.

4. **전문 추론 모델 사용**
   - 이 옵션을 켜면 분석 시 일반 모델 대신 전문 추론 모델을 사용합니다.
   - 긴 철학·과학·경제 지문처럼 구조가 복잡한 경우에 유용합니다.

5. **추론 강도**
   - 모델이 어느 정도 깊게 추론할지 정합니다.
   - 낮게 설정하면 빠르고 저렴할 수 있지만 분석이 단순해질 수 있습니다.
   - 높게 설정하면 더 깊게 분석할 수 있지만 비용과 시간이 늘어날 수 있습니다.

6. **서술형 채점 모델**
   - 서술형 답안을 AI로 채점할 때 사용할 모델입니다.
   - 서술형 채점은 반드시 최고 성능 모델이 필요하지 않을 수 있으므로, 저비용 모델을 따로 설정할 수 있게 되어 있습니다.

7. **비용·보안 안내**
   - GitHub Pages 같은 정적 배포 환경에서는 API 키가 브라우저에서 직접 사용됩니다.
   - 앱이 키를 서버에 저장하지는 않지만, 공개 서비스로 운영하려면 서버 프록시를 붙이는 것이 안전합니다.
   - 다른 사람에게 배포할 때는 개인 API 키를 코드에 직접 넣으면 안 됩니다.

## 5. API 키 입력과 주의사항

이 앱은 사용자의 OpenAI API 키를 이용해 분석을 실행합니다. API 키가 없으면 데모 데이터는 볼 수 있지만, 실제 지문 분석은 실행할 수 없습니다.

### API 키 입력 순서

1. 오른쪽 위 `메뉴`를 엽니다.
2. `OpenAI API Key` 칸에 키를 붙여넣습니다.
3. `키 적용` 버튼을 누릅니다.
4. 화면에 `키 인식됨` 표시가 뜨는지 확인합니다.
5. 지문을 입력하고 `분석하기`를 누릅니다.

### API 키가 인식되지 않을 때

다음 사항을 확인하세요.

- 키 앞뒤에 공백이 들어가지 않았는지 확인합니다.
- `키 적용` 버튼을 눌렀는지 확인합니다.
- 데모 모드가 켜져 있지는 않은지 확인합니다.
- 브라우저를 강력 새로고침합니다.
- 이전 버전 폴더에서 서버를 켜고 있지는 않은지 확인합니다.

특히 로컬 서버 주소가 항상 `127.0.0.1:8123`으로 같기 때문에, 실제로 어떤 폴더에서 서버를 켰는지 헷갈릴 수 있습니다. 새 버전을 받았다면 기존 서버를 `Ctrl + C`로 끈 뒤, 새 버전 폴더에서 다시 서버를 실행해야 합니다.

## 6. 지문 입력 방법

왼쪽 영역은 지문 입력 및 본문 확인 영역입니다. 여기에 분석할 국어 지문을 넣습니다.

### 직접 입력한 지문

직접 타이핑한 지문이나 이미 문단이 정리된 지문은 그대로 붙여 넣어도 됩니다. 문단 사이에는 빈 줄을 하나 넣는 것이 좋습니다.

예시:

```text
첫 번째 문단 내용입니다.

두 번째 문단 내용입니다.

세 번째 문단 내용입니다.
```

### OCR 지문

OCR로 가져온 지문은 다음과 같은 문제가 자주 있습니다.

- 문단 내부에서 줄이 강제로 끊김
- 띄어쓰기가 이상함
- 붙어쓰기가 많음
- 단어 사이가 이상하게 벌어짐
- 원래 한 문장인데 여러 줄로 나뉨

이 경우 먼저 문단 사이에 빈 줄을 넣어 주세요. 문단 경계는 사용자가 직접 잡아 주는 것이 가장 안정적입니다. 그 뒤 `OCR 정리` 또는 `AI 정돈`을 사용합니다.

## 7. OCR 정리 기능

`OCR 정리`는 AI를 사용하지 않는 간단한 정리 기능입니다. 문단 내부의 강제 줄바꿈을 이어 붙이고, 문단 사이 빈 줄은 유지하려고 시도합니다.

### 언제 사용하면 좋은가

- OCR 결과가 문장 중간마다 줄바꿈되어 있을 때
- 한 문단이 여러 줄로 쪼개져 있을 때
- 문단 사이 빈 줄은 이미 사용자가 넣어 둔 상태일 때

### 사용 순서

1. OCR 지문을 붙여넣습니다.
2. 문단 사이에 빈 줄을 넣습니다.
3. `OCR 정리` 버튼을 누릅니다.
4. 문단 내부 줄바꿈이 자연스럽게 이어졌는지 확인합니다.

`OCR 정리`는 빠르고 비용이 들지 않지만, 띄어쓰기나 문장 의미를 깊게 고치지는 않습니다.

## 8. AI 정돈 기능

`AI 정돈`은 OpenAI API를 사용하여 OCR 지문의 띄어쓰기, 붙어쓰기, 줄바꿈을 더 자연스럽게 정리하는 기능입니다.

### 언제 사용하면 좋은가

- OCR 결과의 띄어쓰기가 심하게 깨졌을 때
- `곧철학`, `볼수있다`, `설수있다`처럼 붙어쓰기 오류가 많을 때
- 문단 내부 문장 흐름이 어색할 때
- 분석 전에 지문 자체를 읽기 좋게 다듬고 싶을 때

### 주의할 점

AI 정돈은 지문을 새롭게 해석하거나 요약하는 기능이 아닙니다. 원문 의미를 바꾸지 않고 표기와 줄바꿈만 정리하도록 설계되어 있습니다. 그래도 AI가 개입하는 기능이므로, 시험 지문처럼 원문 보존이 중요한 경우에는 정돈 후 원문과 비교해 보는 것이 좋습니다.

## 9. 분석하기

지문을 넣고 설정을 마쳤다면 `분석하기` 버튼을 누릅니다.

분석 과정에서는 다음과 같은 단계 안내가 표시됩니다.

- 문단 분리
- 핵심 주장 탐색
- 형광펜 설계
- 마인드맵 구성
- 학습 포인트 정리

분석이 오래 걸리는 경우에도 화면이 멈춘 것이 아닐 수 있습니다. 특히 긴 지문이나 전문 추론 모델을 사용할 때는 마지막 단계에서 시간이 더 걸릴 수 있습니다. 이때 앱은 `분석 중이니 잠시만 기다려 주세요`와 같은 안내를 표시합니다.

분석이 끝나면 오른쪽 탭에 결과가 채워집니다.

## 10. 요약 탭

요약 탭은 전체 내용을 가장 먼저 파악하는 공간입니다. 여기에는 다음 항목이 표시됩니다.

1. **분야 태그**
   - 예: 철학, 과학, 경제, 기술, 문학 등

2. **전체 제목 또는 핵심 주제**
   - 지문이 무엇을 설명하는지 한 줄로 보여 줍니다.

3. **전체 요약**
   - 지문의 중심 내용을 압축해서 보여 줍니다.

4. **읽는 법**
   - 이 지문을 어떤 순서로 읽어야 하는지 안내합니다.
   - 예를 들어 “개념 정의 → 차이 구분 → 역전 논리 → 결론”처럼 독해 흐름을 제시합니다.

요약 탭은 지나치게 화려한 형광펜 효과를 넣지 않고, 가독성 좋은 카드형 텍스트로 유지됩니다. 목적은 예쁘게 꾸미는 것보다 지문 전체의 방향을 빠르게 잡는 것입니다.

## 11. 난이도 평가

분석 결과에는 난이도 평가가 포함됩니다. 난이도는 단순히 AI가 느낌으로 말하는 것이 아니라, 5개 기준을 나누어 평가하도록 설계되어 있습니다.

대표 기준은 다음과 같습니다.

1. **개념 밀도**
   - 핵심 개념이 얼마나 많이 등장하는지 평가합니다.

2. **문장 난도**
   - 문장 길이, 추상어, 부정 표현, 관념적 표현의 정도를 평가합니다.

3. **구조 복잡도**
   - 글의 전개가 단순 설명인지, 비교·대조와 전환이 많은지 평가합니다.

4. **추론 요구도**
   - 직접 진술을 찾는 수준인지, 앞뒤 개념을 연결해야 하는지 평가합니다.

5. **선지 함정 가능성**
   - 문제로 만들었을 때 오답 선지가 얼마나 그럴듯하게 만들어질 수 있는지 평가합니다.

종합 난이도는 5점 만점으로 표시됩니다. 예를 들어 `4.2/5`는 상당히 어려운 편이지만, 앱에서는 점수 구간에 따라 `중상`, `상`처럼 더 자연스러운 등급으로 보이도록 구성되어 있습니다.

## 12. 지문 본문과 형광펜

분석이 끝나면 왼쪽 지문 영역에 형광펜이 표시됩니다. 형광펜은 단순히 어려운 부분에만 칠하는 기능이 아닙니다. 이 앱에서 형광펜은 “쓸데없는 부분은 덜 보고, 중요한 구조만 빠르게 보는 기능”입니다.

### 형광펜 종류

- **핵심 주장**
  - 필자나 사상가의 중심 견해
- **근거**
  - 주장이나 개념을 뒷받침하는 부분
- **비교·대조**
  - A와 B의 차이를 보여 주는 부분
- **개념 정의**
  - 핵심어의 의미를 설명하는 부분
- **예시**
  - 추상 개념을 구체화하는 사례
- **주의·반론**
  - 오해하기 쉬운 부분, 선지 함정이 될 수 있는 부분
- **부연·균형**
  - 핵심 주장을 보완하거나 균형을 잡는 설명

### 형광펜이 겹치는 경우

같은 구절은 여러 역할을 동시에 가질 수 있습니다. 예를 들어 어떤 문장은 핵심 주장이면서 동시에 비교·대조의 기준일 수 있습니다. 따라서 앱은 형광펜이 겹칠 수 있도록 설계되어 있습니다.

### 형광펜 호버 설명

형광펜 위에 마우스를 올리면 설명이 뜹니다. 이 설명은 단순히 “핵심 주장입니다”라고 말하는 것이 아니라, 다음 내용을 보여 주는 것을 목표로 합니다.

- 이 구절이 지문에서 어떤 역할을 하는가
- 무엇의 근거인가
- 어떤 개념과 연결되는가
- 무엇과 대조되는가
- 어떤 선지 함정으로 바뀔 수 있는가
- 왜 이 구절을 놓치면 독해가 흔들리는가

### 형광펜 클릭

형광펜을 클릭하면 더 자세한 설명 카드가 열립니다. 이 카드는 해당 구절을 중심으로 구조적 의미를 확인하는 용도입니다.

## 13. 구조 탭

구조 탭은 글의 전개 흐름을 보여 줍니다. 문단별로 다음 정보가 표시됩니다.

- 문단 번호
- 소제목
- 문단 역할
- 핵심 내용
- 관련 문단 버튼

예를 들어 철학 지문이라면 다음과 같은 흐름이 나타날 수 있습니다.

```text
1문단: 상황 내존재 개념 제시
2문단: 한계 상황의 정의
3문단: 한계 상황의 의미 전환
4문단: 초월자 개념 소개
5문단: 암호와 실존 회복
```

문단 버튼을 누르면 해당 문단 위치로 이동하거나, 해당 문단이 강조됩니다. 구조 탭은 지문을 다시 읽을 때 “지금 내가 어디를 읽고 있는지”를 잡아 주는 지도 역할을 합니다.

---

## 14. 비교·대조

비교·대조 항목은 국어 문제에서 자주 출제되는 부분입니다. 이 앱은 단순히 `A`와 `B`라고 표시하는 것이 아니라, 실제 비교되는 개념명을 보여 주도록 설계되어 있습니다.

예를 들어 다음과 같이 표시됩니다.

```text
언어와 표상 불가능
↔
암호를 통한 비대상적 인식
```

각 비교 항목에는 다음 정보가 포함될 수 있습니다.

- 비교 기준
- 왼쪽 개념
- 오른쪽 개념
- 두 개념이 어떻게 다른지
- 지문 어디에서 이 차이가 드러나는지
- 문제 선지에서 어떻게 바뀌어 나올 수 있는지

비교·대조는 특히 `A와 달리 B는`, `A와 B의 공통점으로 적절한 것은`, `A와 B에 대한 설명으로 적절하지 않은 것은` 같은 문제의 기반이 됩니다.

---

## 15. 개념 탭

개념 탭은 지문 속 핵심어를 정리하는 공간입니다. 각 개념은 한 행 또는 카드 형태로 표시되며, 다음 요소를 가질 수 있습니다.

- 개념명
- 쉬운 설명
- 지문 속 의미
- 출처 문단
- 선택적 예시

예시는 강제하지 않습니다. 억지 예시는 오히려 지문 이해를 방해할 수 있기 때문입니다. AI가 적절하다고 판단할 때만 예시가 들어갑니다.

개념 설명을 클릭하면 본문에서 해당 개념이 등장한 부분이 잠시 강조됩니다. 강조는 약 3초 뒤 사라지도록 설계되어 있습니다. 이 기능은 개념 목록과 원문을 연결해서 읽기 위한 것입니다.

## 16. 헷갈리는 문장

헷갈리는 문장 영역은 지문에서 학생들이 어려워할 만한 문장을 골라 보여 줍니다.

각 항목에는 다음 정보가 들어갑니다.

- 원문 문장
- 쉬운 설명
- 출제 포인트 또는 주의할 점

예전 방식처럼 `어려운 이유`, `출제 포인트`를 불필요하게 펼쳐 놓기보다는, 핵심은 `쉽게` 이해하는 데 두었습니다. 문장을 클릭하면 해당 문장이 본문 어디에 있는지 확인할 수 있도록 강조 기능이 연결됩니다.

## 17. 마인드맵 사용법

마인드맵은 지문의 구조를 시각적으로 보는 공간입니다. 중심 노드에서 주요 개념이 가지처럼 뻗고, 다시 세부 개념이 이어지는 방식입니다.

### 기본 조작

- 노드 드래그: 노드 위치를 직접 옮길 수 있습니다.
- 빈 공간 드래그: 화면 전체를 이동할 수 있습니다.
- 휠클릭 드래그: 마우스 휠 버튼을 누른 채 화면을 이동할 수 있습니다.
- 마우스 휠: 확대·축소할 수 있습니다.
- `＋` 버튼: 확대합니다.
- `−` 버튼: 축소합니다.
- `초기화` 버튼: 배치를 초기 상태로 되돌립니다.

### 모바일 조작

모바일이나 태블릿에서는 손가락으로 빈 공간을 밀어 화면을 이동할 수 있습니다. 확대·축소는 버튼을 사용하는 것이 가장 안정적입니다.

### 마인드맵이 잘려 보일 때

마인드맵은 지문 내용에 따라 노드가 많아질 수 있습니다. 이 경우 다음 방법을 사용하세요.

1. `−` 버튼을 눌러 축소합니다.
2. 빈 공간을 드래그해 중심을 이동합니다.
3. 휠클릭 드래그로 화면을 끌어옵니다.
4. 배치가 너무 꼬이면 `배치 초기화`를 누릅니다.

## 18. 문제 생성

분석이 끝나면 `문제 제작` 버튼으로 문제를 만들 수 있습니다. 문제 생성은 시간이 꽤 걸릴 수 있습니다. 특히 5지선다 문제는 선지를 먼저 만드는 것이 아니라, 출제 의도와 정답·오답 이유를 먼저 설계한 뒤 선지를 구성하도록 프롬프트가 작성되어 있습니다. 그래서 마지막 단계에서 시간이 더 걸릴 수 있습니다.

문제 생성 중에는 `문제를 제작 중이에요`라는 안내가 표시됩니다. 화면이 멈춘 것처럼 보여도 API 요청이 진행 중일 수 있으니 잠시 기다려 주세요.

### 문제 유형

1. 5지선다형
2. OX형
3. 서술형

## 19. 5지선다형 풀이

5지선다형은 실제 국어 문제처럼 사용자가 먼저 풀도록 되어 있습니다. 정답과 해설은 처음부터 보이지 않습니다.

### 풀이 순서

1. 문제를 읽습니다.
2. 필요하면 `<보기>`를 읽습니다.
3. ①~⑤ 선지 중 하나를 선택합니다.
4. 선택한 선지는 색으로 강조됩니다.
5. `채점 및 정오 보기`를 누릅니다.
6. 정답, 오답 이유, 발췌 근거가 열립니다.

### 발췌 근거가 처음부터 보이지 않는 이유

발췌 근거는 사실상 정답 위치를 알려 줄 수 있습니다. 그래서 문제를 풀기 전에는 숨기고, 채점 후에만 보여 주도록 되어 있습니다.

### 좋은 5지선다 문항의 기준

이 앱의 문제 생성 프롬프트는 다음 기준을 따르도록 되어 있습니다.

- 정답만 지나치게 구체적이면 안 됨
- 오답이 너무 짧거나 허술하면 안 됨
- 모든 선지는 지문 속 표현을 적절히 활용해야 함
- 정확히 읽지 않으면 모든 선지가 그럴듯해 보여야 함
- 단순 내용 일치뿐 아니라 비교·대조, 추론, 보기 적용 문제가 포함되어야 함
- 적절한 것, 적절하지 않은 것, 관점 비교, 보기 적용, 이유 추론 등 다양한 기출형 표현을 활용해야 함

## 20. OX 문제 풀이

OX 탭은 문제를 카드처럼 크게 나누기보다 한눈에 여러 문항을 볼 수 있는 목록형으로 구성되어 있습니다.

### 사용법

1. 각 문장 오른쪽의 O 또는 X를 선택합니다.
2. 선택한 항목은 강조됩니다.
3. 각 문항의 `채점` 버튼으로 개별 채점할 수 있습니다.
4. 또는 `전체 채점` 버튼으로 한 번에 채점할 수 있습니다.
5. 틀린 문제는 정오 해설을 확인합니다.
6. 다시 풀고 싶으면 `다시 시도` 또는 전체 다시 풀기 기능을 사용합니다.

OX는 빠르게 지문 이해도를 확인하는 용도입니다. 단순 사실 확인뿐 아니라, 개념의 범위나 논리 관계를 살짝 바꾸는 방식으로 출제될 수 있습니다.

## 21. 서술형 문제 풀이

서술형 탭에서는 직접 답안을 작성할 수 있습니다. 서술형은 단답형이 아니라, 개념 적용·이유 설명·전개 방식 설명·비교 서술 등을 훈련하기 위한 기능입니다.

### 사용법

1. 서술형 문제를 읽습니다.
2. 답안 입력 칸에 자신의 답을 씁니다.
3. 개별 문항만 채점하려면 해당 문항의 `AI 채점` 버튼을 누릅니다.
4. 전체를 한 번에 채점하려면 `전체 AI 채점`을 누릅니다.
5. 채점 전에는 API 키를 사용한다는 안내가 표시됩니다.
6. AI가 사용자의 답안, 기준 답안, 채점 기준을 함께 보고 피드백을 제공합니다.

### 서술형 채점에서 AI가 보는 것

AI 채점에는 다음 정보가 전달됩니다.

- 문제
- 사용자가 작성한 답안
- 기준 답안
- 채점 기준
- 필요한 경우 지문 요약 또는 관련 분석 정보

서술형 채점은 토큰을 사용할 수 있으므로, 여러 문제를 따로따로 채점하는 것보다 전체 채점을 사용하는 편이 더 효율적일 수 있습니다.

## 22. 문제별 AI 질문

문제를 풀다가 납득이 안 되는 부분이 있을 수 있습니다. 이때 각 문항의 `AI에게 질문하기` 버튼을 사용할 수 있습니다.

예를 들어 다음과 같은 질문을 할 수 있습니다.

- 왜 ③번이 틀린 선지인가요?
- ②번과 ④번이 헷갈리는데 차이를 설명해 주세요.
- 이 문제는 지문의 어느 부분을 묻고 있나요?
- 보기의 사례를 어떻게 지문 개념에 적용해야 하나요?
- 제 답이 왜 틀렸는지 쉽게 설명해 주세요.

질문 기능은 문제 풀이를 단순 채점에서 끝내지 않고, 오답을 이해하는 과정으로 이어 주는 역할을 합니다.

## 23. 메모 탭과 드래그 설명

지문에서 이해가 안 되는 구절을 드래그한 뒤 메모 탭으로 이동하면 AI 설명을 요청할 수 있습니다.

### 사용법

1. 왼쪽 지문 영역에서 문장이나 구절을 드래그합니다.
2. 메모 탭으로 이동합니다.
3. `선택 구절 분석` 또는 유사한 분석 버튼을 누릅니다.
4. 분석 중 표시가 뜨는지 확인합니다.
5. AI가 해당 구절을 쉽게 설명하고, 지문 속 역할을 정리합니다.

메모는 단순한 필기가 아니라 “내가 헷갈린 구절을 저장하고 다시 설명받는 공간”입니다. 어려운 철학 지문이나 과학 지문을 반복해서 공부할 때 유용합니다.

## 24. 저장 탭

저장 탭에서는 현재 분석 결과를 저장하고, 이전 분석을 다시 불러올 수 있습니다.

### 저장되는 내용

- 원문 지문
- 정돈된 지문
- 분석 결과
- 형광펜 정보
- 마인드맵 정보
- 문제 생성 결과
- 사용자의 문제 선택 상태
- 메모

브라우저의 localStorage를 사용하므로 같은 브라우저에서는 다시 열 수 있습니다. 다만 브라우저 데이터 삭제, 시크릿 모드, 다른 기기에서는 저장 내용이 유지되지 않을 수 있습니다.

### 샘플과 내 지문 전환

샘플을 눌러 앱의 예시를 확인한 뒤 `내 지문`으로 돌아오면 기존 작업 상태가 유지되도록 되어 있습니다. 즉, 샘플을 잠깐 본다고 해서 분석 결과가 날아가면 안 됩니다.

### 새 분석 노트

완전히 새로운 지문을 분석하고 싶다면 `새 분석 노트` 버튼을 사용합니다. 이 버튼은 현재 작업을 초기화하고 새 입력 상태로 돌아가게 합니다. 기존 작업을 보존하고 싶다면 먼저 저장한 뒤 새 노트를 만드세요.

## 25. 오류와 로그 확인

오류가 발생하면 왼쪽 아래에 오류 토스트가 표시됩니다.

### 일반 안내와 오류의 차이

- 일반 안내: 3초 뒤 자동으로 사라집니다.
- 오류 메시지: 자동으로 사라지지 않습니다.

오류는 원인을 확인해야 하므로 3초 뒤 사라지지 않도록 설계되어 있습니다. 오류 카드의 `로그 보기`를 누르면 더 자세한 정보를 볼 수 있습니다. 확인 후 직접 `×` 버튼으로 닫으면 됩니다.

### 자주 발생하는 오류

1. **API 키 없음**
   - 메뉴에서 API 키를 입력하고 `키 적용`을 눌러야 합니다.

2. **모델명 오류**
   - 입력한 모델명이 현재 계정에서 사용할 수 없는 모델일 수 있습니다.

3. **JSON 파싱 오류**
   - 모델 응답이 스키마를 벗어난 경우 발생할 수 있습니다.
   - 다시 시도하거나 더 안정적인 모델을 사용하세요.

4. **네트워크 오류**
   - 인터넷 연결 또는 OpenAI API 요청 문제일 수 있습니다.

5. **캐시 문제**
   - 이전 버전 JavaScript가 남아 있으면 이상 동작할 수 있습니다.
   - `Ctrl + F5`로 강력 새로고침하세요.

## 27. GitHub Pages 배포 방법

이 저장소에는 GitHub Pages 배포용 워크플로가 포함될 수 있습니다. 일반적으로 다음 파일이 있으면 GitHub Actions를 통해 배포할 수 있습니다.

```text
.github/workflows/pages.yml
.nojekyll
```

### 배포 순서

1. 저장소에 파일을 push합니다.
2. GitHub 저장소 페이지로 이동합니다.
3. `Settings`로 들어갑니다.
4. 왼쪽 메뉴에서 `Pages`를 선택합니다.
5. `Build and deployment`의 Source를 `GitHub Actions`로 설정합니다.
6. Actions 탭에서 배포 워크플로가 실행되는지 확인합니다.
7. 배포가 끝나면 다음 형식의 주소로 접속합니다.

```text
https://satasun5.github.io/korean-analyzer/
```

### 배포 후 주의사항

GitHub Pages는 정적 호스팅입니다. 즉, 서버에서 API 키를 숨겨 주지 않습니다. 사용자가 직접 자기 API 키를 입력하는 개인용 앱으로는 사용할 수 있지만, 공개 서비스처럼 여러 사람에게 제공하려면 서버 프록시를 따로 만들어야 합니다.

## 30. 스트리밍 기능에 대하여

스트리밍 분석 기능은 실험 폴더에 분리되어 있습니다. 일반 분석은 안정성을 우선하여 전체 JSON 응답을 받은 뒤 렌더링하는 방식입니다. 스트리밍은 실시간으로 글자가 생성되는 느낌을 줄 수 있지만, Structured Outputs와 결합할 때 응답 중간 상태가 완전한 JSON이 아니기 때문에 충돌 가능성이 있습니다.

따라서 현재는 다음 파일에 실험 형태로 분리되어 있습니다.

```text
experimental/streaming-analysis.js
experimental/README-streaming.md
```

스트리밍을 본 기능에 합치려면 다음 방식이 권장됩니다.

1. 진행 메시지와 단계 설명만 스트리밍한다.
2. 실제 분석 결과 JSON은 마지막에 한 번에 파싱한다.
3. 마인드맵과 문제 UI는 완성된 JSON을 받은 뒤 렌더링한다.

이렇게 하면 실시간 느낌과 안정성을 동시에 확보할 수 있습니다.

## 33. 마지막 안내

반짝국어는 완성형 상용 서비스라기보다는, 국어 지문 분석 학습을 위한 실험적 웹앱입니다. 지문 구조화, 형광펜, 마인드맵, 문제 생성이라는 여러 기능을 한 화면에 모았기 때문에, 실제 사용하면서 계속 레이아웃과 프롬프트를 다듬는 것이 중요합니다.

특히 국어 문제 생성은 단순히 정답을 만드는 것보다 “정답처럼 보이는 오답”을 만드는 것이 훨씬 어렵습니다. 따라서 모델, 프롬프트, 지문 길이에 따라 문제 품질이 달라질 수 있습니다. 좋은 결과가 나오지 않으면 전문 추론 모델을 사용하거나, 문제 생성 프롬프트를 더 엄격하게 조정하는 것이 좋습니다.

이 앱의 목표는 사용자가 지문을 외우게 만드는 것이 아니라, 글의 구조를 보고, 개념의 관계를 파악하고, 선지가 어디서 함정을 만드는지 직접 확인하게 만드는 것입니다.

- 메모마다 이어 질문 입력창과 빠른 질문 버튼을 추가했습니다. 이전 메모 대화 맥락을 이어서 AI에게 다시 질문할 수 있습니다.

- 메모 질문용 AI 호출 함수와 프롬프트를 추가했습니다.
